# Lytos

[![npm](https://img.shields.io/npm/v/lytos-cli)](https://www.npmjs.com/package/lytos-cli)

> A human-first method for working with AI agents. Context, not costumes.

**[lytos.org](https://lytos.org)** · **[CLI on npm](https://www.npmjs.com/package/lytos-cli)**

---

Your AI resets every session. Lytos gives it persistent memory, rules, and the right context — in Markdown files, in your Git repo. One command to set up. Works with Claude Code, Cursor, Codex, or any LLM. No vendor lock-in.

> *"Agile structured human collaboration. Lytos structures collaboration with AI."*

## Get started

```bash
npm install -g lytos-cli
lyt init
```

Then open your AI tool and say: **"Read .lytos/manifest.md and help me work on this project."**

## The 5 pillars

| Pillar | What it is | File |
|--------|-----------|------|
| **Intent** | Your project's constitution — the why | `manifest.md` |
| **Design** | Reusable procedures — the how | `skills/` |
| **Standards** | Quality criteria — the non-negotiable | `rules/` |
| **Progress** | Issues and sprint — what's moving | `issue-board/` |
| **Memory** | Persistent knowledge — what we learned | `memory/` |
